Binary Clock

This Project creates a nice Binary Clock using the Atmega48a, some 3D Printing, a DCF77, and as well a CNC-Machine.

The Main Objective

I had the task of working on this specific lab experiment throughout the entire lecture period of the semester. Together with a partner, we had to design two unique circuit boards for a binary clock, manufacture them using the CNC milling machine, and then assemble them with the necessary components. At any point during the semester, we could present the finished circuit board and perform all the required tests, as outlined in the instructions, after coordinating with the examiners.

More concrete steps:

  1. Design of the circuit diagram using the provided components.
  2. Test the circuit diagram with the Schematic Rule Check.
  3. Design a PCB layout based on the circuit diagram.
  4. Test the layout with the Design Rule Check.
  5. Print the layout on paper and test again.
  6. Schedule an appointment with the student assistant.
  7. Create the PCB under the supervision of the student assistant.
  8. Repeat the previous step if necessary.
  9. Assemble the components on the PCB.
  10. Solder the components.
  11. Test connections using a multimeter.
  12. First power-up and debugging.

What we had in Plan

My friend and I had some other plans...

We bought an DCF77 from Aliexpress and tried to make the Binary Clock able to receife the Time over the Air.

This was even possible without WIFI. In Germany we have a Radio Frequence of 77.5kHz which sends out the time in a one-minute intervall. More can be read here: https://en.wikipedia.org/wiki/DCF77

The final result:

The final result can be seen on the Github Page auf our project: https://github.com/max0l/BinaryClock. There are also all files to recreate this project.

The new Dezentrale Door-Locking-System

How it all Started

The Hackspace where I'm a member of will move soon.

But, because we are moving to a not so nice part of the city, we needed a new Door-Locking-System. A team, with a friend and me as the Team leaders, were tasked to develop a new System for the new Hackspace Location.

I have always been interested in the mechanics of NFC, and this Project taught me a lot. We needed to think of a System that works reliably, fast and is secure. For this, we met with some other people from the Hackspace to discuss it.

The Result was, that we will use NFC-Cards and also make it possible to unlock the doors over an SSH connection. This would insure two ways if one failed.

How it's going

Unfortunatly, since of now, it has not been finally build. This is because we didn't moved yet. But there are some Proto types.

What is the "Dezentrale" You ask?

The dezentrale e.V is an independent non-profit association that runs a hackspace in the western part of Leipzig. A hackspace (also referred to here as "space") is a place for exchanging ideas and working on projects, which tend to be related to technology but don’t have to be. Occasionally, activities like sewing might take place as well. The space is a community-managed location where people with an interest in technology can come together to learn from each other, work on projects, or discuss (internet policy) topics. Perhaps you’ve heard of the Chaos Computer Club: We are the local Chaos in Leipzig. (We are an experience-sharing group of the Chaos Computer Club). Endlessh

A few days ago, I set up Endlessh-go (aka SSH-Tarpit). It's quite amusing to see how SSH scanners fall for it. Brief explanation: Endlessh listens on an open port, and when someone tries to log in via SSH, nothing happens. However, the client is not disconnected, which causes them to be 'trapped' and slowed down indefinitely. You can also integrate Endlessh nicely into Grafana. It looks something like this: Just for fun, I looked up a German IP in the Whois that stood out particularly and sent an email to the registered abuse contact. The IP belonged to a hosting provider. Let’s see what comes out of it. I also searched for the IP on CrowdSec and quickly found it had already been flagged before.
