About

Here I will discuss which and why I used which HTML Element and will elaborate on design decisions. You can view my development progress on my Github Repo.

The main Idea

The main idea of my Website is to look minimalistic, modern with a touch of Computer Science and Retro Gaming.

This is done by using a more Code like looking font and a VHS Screen video that acts as the background to add this old VHS noise and feeling. I also tried to choose a background Video which is very suptile.

The Design Elements

Structural HTML Elements

<div class="content">
I used the <div class="content"> to separate the video, which functions as the background, and the contents. This will ensure that when I can apply CSS rules on the content more easily.
<header>
The <header> is used to describe the Navigation Menu at the top of the website. It is fixed at the top of the Website.
<nav>
The <nav> hold the <ul> with each <li> holding a link to a site. The <nav> is used because it is the semantic tag exactly used for this propose.
<main>
<main> is used to describe the main contents of the site. It has an ID=top because I wanted the functionality that If you are at the bottom of the site, you could easily get to the top again. This is done by creating an href that links to the #top of the site.
<articel>
The article tag is used to define the main article. Sometimes I used a <aside> which does not have to be in the main article.
<section>
A <section> is used to structure an article in itself. Here I used other Elements like <h1>, <ul>, <p> or <img>.
<footer>
Similar to the <header>. It was used to create a footer that showed the "back to top link" as well some other links you usually find at the footer.
<address>
The Address was used to surprise write about my address. I used the <br> there to make the formatting better (as learned in the Lecture).

Other Design Decisions

There are a few other decisions I have made during this assignment which I also want to elborate.

Folder Structure

The folder structure shows that I have a main folder with, index.html, resume.html, contact.html and about.html. This is all in the main Folder because I think it is more easy to understand, than having for every html file in the main an extra folder and than named every page index.html.

For the Portfolio it is something esle because there are many Subfolders. This would blow up the Navigation (if no Dropdown). So the Portfolio has an index.html to navigate from there to the Subpages. Also, my file Strucutr is suggested by this resource, which was linked at the Oblig description.
