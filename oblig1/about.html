About

Here I will discuss which and why I used which HTML Element and will elaborate on design decisions. You can view my development progress on my Github Repo.

The main Idea

The main idea of my Website is to look minimalistic, modern with a touch of Computer Science and Retro Gaming.

This is done by using a more Code like looking font and a VHS Screen video that acts as the background to add this old VHS noise and feeling. I also tried to choose a background Video which is very suptile.

The Design Elements

Structural HTML Elements

<div class="content">

I used the <div class="content"> to separate the video, which functions as the background, and the contents. This will ensure that I can apply CSS rules on the content more easily.

<header>

The <header> is used to describe the Navigation Menu at the top of the website. It is fixed at the top of the Website.

<nav>

The <nav> holds the <ul> with each <li> containing a link to a site. The <nav> is used because it is the semantic tag exactly used for this purpose.

<main>

The <main> is used to describe the main content of the site. It has an ID=top because I wanted the functionality that if you are at the bottom of the site, you can easily get to the top again. This is done by creating an href that links to the #top of the site.

<article>

The article tag is used to define the main article. Sometimes I used an <aside>, which does not have to be in the main article.

<section>

A <section> is used to structure an article in itself. Here I used other elements like <h1>, <ul>, <p>, or <img>.

<footer>

Similar to the <header>, it was used to create a footer that showed the "back to top" link as well as some other links you usually find at the footer.

<address>

The <address> was used to surprise write about my address. I used the <br> there to make the formatting better (as learned in the lecture).

Semantic HTML Elements

<strong>

I used the <strong> tag to emphasize that I am the FIRST child of my parents, which I mentioned on the landing page.

<em>

I used the <em> tag when I described the touch of Computer Science, which is not easily describable.

<cite>

The <cite> tag was used to reference a quoted work.

<del>

This tag was used to cross out a bad word and "replace" it.

<blockquote>

This was used for a longer quote I took from the Dezentrale website.

<tr>

I used a table for my resume. I have a really nice-looking resume latex template and I tried to reproduce it in HTML.

<sup>

The <sup> was used to superscript the "st" in my birthday.

<abbr>

I used the <abbr> to explain an abbreviation or acronym.

<dfn>

<dfn> was used to explain what the Dezentrale is.

<figure>

A figure was used on the doorsystem.html page to explain a diagram.

Other Design Decisions

There are a few other decisions I made during this assignment that I want to elaborate on.

Folder Structure

The folder structure shows that I have a main folder containing index.html, resume.html, contact.html, and about.html. This is all in the main folder because I think it is easier to understand than having an extra folder for every HTML file in the main directory and then naming every page index.html.

For the Portfolio, it is something else because there are many subfolders. This would blow up the Navigation (if no Dropdown). So, the Portfolio has an index.html to navigate from there to the subpages. My file structure is suggested by this resource, which was linked at the Oblig description.
